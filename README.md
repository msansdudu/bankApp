# BankApp

Прототип (симулятор :)) банковского приложения с бэкендом на Spring Boot и мобильным клиентом на React Native.

## Основные возможности
- **Авторизация**: Безопасный вход и регистрация пользователей.
- **Дашборд**: Обзор банковских счетов и истории транзакций.
- **Детали карты**: Информация по каждому счету.
- **Создание счета**: Создание нового счета с выбором валюты.
- **Переводы**: Внутренние переводы между счетами с валидацией баланса и валюты.
- **Уведомления**: Просмотр системных и транзакционных уведомлений.
- **Темы**: Поддержка светлого и темного режимов оформления.

## Стек технологий
- **Backend**: Java, Spring Boot, Spring Security (JWT), PostgreSQL.
- **Frontend**: React Native (Expo), Axios, React Navigation, Context API.

## Инструкция по запуску

### Бэкенд (Spring Boot)
1. Перейдите в директорию `bank-backend/`.
2. Запустите базу данных **PostgreSQL**.
3. Настройте параметры подключения к БД в файле `src/main/resources/application.properties`.
4. Запустите проект:
   ```bash
   ./mvnw spring-boot:run
   ```

### Фронтенд (React Native)
1. Перейдите в директорию `frontend/`.
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Укажите ваш локальный IP-адрес в файле `shared/api/config.js` (необходимо для подключения с реального устройства или эмулятора):
   ```javascript
   export const API_URL = 'http://ВАШ_IP:8080/api';
   ```
4. Запустите приложение:
   ```bash
   npm start
   ```
5. Далее по инструкции в консоли

## Технические подробности проекта:

### Бэкенд (Spring Boot)
- **Безопасность**: Реализована аутентификация на основе **JWT (JSON Web Tokens)**. Пароли пользователей хранятся в зашифрованном виде (BCrypt).
- **Слой данных**: Использование **Spring Data JPA** для работы с PostgreSQL. Реализована логика транзакций для безопасного перевода средств между счетами (атомарность операций).
- **Валидация**: Проверка баланса, соответствие валют, существование счетов и централизованная обработка исключений через `@ControllerAdvice`.
- **REST API**: Стандартизированные эндпоинты для управления пользователями, счетами и транзакциями.

### Фронтенд (React Native / Expo)
- **Архитектура**: Приложение построено по модульному принципу (близко к архитектуре FSD), разделяя логику на сущности (`entities`), виджеты (`widgets`), страницы (`pages`) и общие утилиты (`shared`).
- **Управление состоянием**: Используется **React Context API** для глобального управления сессией пользователя (`UserContext`) и темами оформления (`ThemeContext`). Это обеспечивает синхронизацию данных и внешнего вида между всеми экранами.
- **Стилизация**: Реализована кастомная система тем (Light/Dark) с использованием динамических стилей. Компоненты получают токены цветов напрямую из контекста темы, что позволяет мгновенно переключать интерфейс без перезагрузки приложения.
- **Работа с API**: Создан единый сервис на базе **Axios** с интерсепторами для автоматического добавления JWT-токенов в заголовки запросов и обработки ошибок.
